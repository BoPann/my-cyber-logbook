# OSI Model

- physical
	- cable, 1 and 0, ethernet cables
- data link
	- mac addr, NIC
- network
		- routing, reassembly 
		- ip, OSPF, RIP
- transport 
	- designed with reliability and guarantee in mind. This protocol reserves a constant connection between the two devices for the amount of time it takes for the data to be sent and received.
	- TCP, UDP 
- session
	- [session vs transport?](https://networkengineering.stackexchange.com/questions/55021/session-layer-vs-transport-layer)
- presenation
	- encryption, format translator
- application
	- user interface
